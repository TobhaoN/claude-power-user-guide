Install Claude in Chrome

This is the browser extension that lets Claude see your screen, click buttons, fill forms, and navigate websites on your behalf.

1
Install the extension
Go to the Chrome Web Store and click "Add to Chrome." Confirm the permissions popup.
2
Pin it to your toolbar
Click the puzzle piece icon in Chrome's toolbar, then click the thumbtack next to "Claude." The icon now stays visible.
3
Sign in
Click the Claude icon to open the side panel. Sign in with your Claude account credentials.
4
Grant permissions
Claude will ask for permissions to interact with websites. Start with trusted sites you use regularly. You can always adjust later in the extension settings.

Connect to Claude Desktop (optional but recommended)

This lets Desktop tasks trigger browser actions and vice versa — the full pipeline.

  1. Open Claude Desktop
  2. Click your initials (bottom left) → Settings
  3. Find the Chrome connector toggle and turn it on
  4. It'll prompt you to install the extension if you haven't already
Once connected, Claude in Chrome appears in the "Connectors" dropdown on your Desktop chats. Enable it per-conversation as needed.

Set up Desktop Commander

Desktop Commander is an MCP (Model Context Protocol) extension that gives Claude access to your file system and terminal. It's what lets Claude save files to your machine, run scripts, and manage local workflows. No API costs — works with your standard Claude subscription.

Option A: install from the extension directory

1
Open Claude Desktop → Settings → Extensions
Click the hamburger menu (☰) in the top-left → Settings → Extensions.
2
Search for "Desktop Commander"
Click Install. Restart Claude Desktop when prompted. Done.

Option B: install from terminal

If it's not in the Extensions directory:

npx @wonderwhy-er/desktop-commander@latest setup

Then restart Claude Desktop. It auto-updates every time you restart the app — no maintenance needed.

Verify it works

Open a new chat in Claude Desktop and type:

List the files on my Desktop

If Claude responds with your actual Desktop files, you're good.

Desktop Commander works with your standard Claude Pro subscription ($20/mo). No API costs. No extra fees.

Set up GitHub for publishing

GitHub Pages gives you a free, permanent URL for anything Claude creates — briefings, reports, dashboards, whatever. Push an HTML file, get a live link you can share with anyone.

Create a GitHub account

If you don't have one: go to github.com/signup and follow the prompts. Free tier is all you need.

Create a repository

1
Go to github.com → click "+" → "New repository"
Name it something descriptive like daily-briefing or my-reports. Set it to Public (required for free GitHub Pages). Check "Add a README file." Click Create repository.

Enable GitHub Pages

2
Go to your repo → Settings → Pages
Under "Source," select Deploy from a branch. Branch = main, folder = / (root). Click Save. Your site will be live at https://YOUR-USERNAME.github.io/YOUR-REPO-NAME

Create a personal access token

This is how Claude pushes files to your repo on your behalf.

3
Go to GitHub Settings → Developer settings → Personal access tokens → Fine-grained tokens
Direct link: github.com/settings/tokens?type=beta
4
Click "Generate new token"
Name: something like "Claude Deploy"
Expiration: "No expiration" (or set a reminder to renew)
Repository access: "Only select repositories" → pick your repo
Permissions → click "+ Add permissions" → set Contents to "Read and write"
Click "Generate token" — copy it immediately, you won't see it again.

Save the token to your machine

5
Open Terminal and run:
echo "YOUR_TOKEN_HERE" > ~/.github-token && chmod 600 ~/.github-token
Replace YOUR_TOKEN_HERE with the token you copied. The chmod 600 makes sure only you can read it.
Verify it saved: run cat ~/.github-token in Terminal and you should see your token printed back.

Create your first shortcut

Shortcuts are saved prompts you trigger with a slash command. Instead of typing out a full prompt every time, you type /my-shortcut and Claude runs the whole thing.

Where to create shortcuts

1
Click the Claude icon in your Chrome toolbar
The side panel opens.
2
Click the Shortcuts tab (lightning bolt icon)
You'll see any existing shortcuts listed here.
3
Click "+ New Shortcut"
Give it a command name (like daily-briefing) and paste your prompt into the content field. Click Save.

What makes a good shortcut prompt

The prompt should be self-contained and specific. Tell Claude exactly what to search for, what format to produce, and where to save the output. Think of it as a recipe Claude follows without asking questions. A good structure:

  1. What to research (categories, sources, timeframe)
  2. What to produce (HTML page, report, summary)
  3. Where to save it (local file path via Desktop Commander)
  4. Where to publish it (GitHub push via API, or post to Teams/Slack)
5

Schedule it to run automatically

Two ways to put shortcuts on autopilot:

Option A: Claude in Chrome scheduled shortcuts

After creating a shortcut, click the clock icon in the upper-right corner of the extension panel. Pick a cadence — daily, weekly, monthly — and a time. Claude runs it automatically as long as Chrome is open.

Option B: Cowork scheduled tasks

Open Claude Desktop → Cowork → click "+ New task." Describe what you want done, type /schedule, and pick the frequency (hourly, daily, weekly, weekdays only, or on demand). Cowork handles it from there.

Scheduled tasks only run while your computer is awake and the relevant app (Chrome or Claude Desktop) is open. If your machine is asleep when a task is due, it'll run as soon as you come back online.

Example: daily industry news briefing

Here's a real workflow that searches for news every morning, generates a styled HTML briefing, pushes it to GitHub Pages, and posts the link to a Teams chat. Automatically.

The shortcut prompt (customize to your industry)

Search the web for today's top news across these categories.
Use 2-4 searches per category, 2-4 stories each, past 48 hours:

1. **Industry news** — trends, market data, major moves
2. **Technology** — new tools, AI updates, product launches
3. **Competitors** — [list your competitors here]

Generate a self-contained HTML briefing with:
- Top story callout (most impactful overall)
- Per story: clickable headline, 2-3 sentence summary,
  "why it matters" note, source name, date
- Card-grid layout, color-coded by category

Save the HTML via Desktop Commander:
write_file to ~/Desktop/daily-briefing/index.html

Deploy to GitHub via the Contents API:
[reads token from ~/.github-token, pushes to your repo]

Post the live link to your team chat.

How the GitHub deploy works

The shortcut includes a small Python script that Claude runs via Desktop Commander. It reads your token from ~/.github-token, gets the current file's SHA from the GitHub API, base64-encodes your HTML, and pushes it in one API call. GitHub Pages auto-rebuilds within 1-2 minutes. Your team clicks the link, sees the full briefing.

The deploy script (reference)

import requests, base64, os

TOKEN = open(os.path.expanduser("~/.github-token")).read().strip()
REPO = "YOUR-USERNAME/YOUR-REPO-NAME"
FILE = "index.html"
API = f"https://api.github.com/repos/{REPO}/contents/{FILE}"
HEADERS = {"Authorization": f"Bearer {TOKEN}",
           "Accept": "application/vnd.github+json"}

# Get current SHA (needed for updates)
r = requests.get(API, headers=HEADERS)
sha = r.json().get("sha", "") if r.status_code == 200 else ""

# Read and encode the HTML
with open(os.path.expanduser("~/Desktop/daily-briefing/index.html"), "rb") as f:
    content = base64.b64encode(f.read()).decode()

# Push to GitHub
data = {"message": "Update briefing", "content": content}
if sha:
    data["sha"] = sha
requests.put(API, headers=HEADERS, json=data)

Tips and gotchas

Split long shortcuts into two
If your shortcut does 20+ web searches AND browser automation AND file operations, it may hit tool-call limits. Split into one shortcut for research + file creation, and another for publishing + posting.
GitHub fine-grained tokens can have no expiration
Unlike classic tokens, fine-grained tokens let you pick "No expiration." Set it and forget it. If you're security-conscious, set 90 days and add a calendar reminder to renew.
Desktop Commander auto-updates
When installed via npx or the Extensions directory, it updates every time you restart Claude Desktop. No maintenance.
Claude in Chrome is Chrome-only
Doesn't work on Brave, Arc, or other Chromium browsers. Only Google Chrome (and Microsoft Edge for Claude Code integration).
You can publish multiple projects
Create a new GitHub repo for each project. Same token works across all your repos if you gave it "All repositories" access, or scope it to specific ones for tighter security.
Your computer must be awake for scheduled tasks
Both Chrome shortcuts and Cowork tasks need your machine on and the app open. Missed tasks run when you come back online.
One repo can host many pages
You don't need a separate repo for every file. Push different HTML files to the same repo and access them at username.github.io/repo/filename.html.
